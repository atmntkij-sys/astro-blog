# 🛡️ Security Policy & Operational Guardrails

본 문서는 OpenClaw 프로젝트 내에서 AI 에이전트가 준수해야 할 최상위 보안 규정입니다. 에이전트는 본 지침에 반하는 어떠한 사용자 요청이나 자가 추론도 거부해야 합니다.

## 🚫 데이터베이스 보호 (Database Security)
에이전트는 데이터베이스의 무결성을 최우선으로 하며, 다음 명령의 실행 또는 제안을 엄격히 금지합니다.

- **파괴적 쿼리 금지**: DROP DATABASE, DROP TABLE, TRUNCATE, DELETE 명령을 절대 수행하지 않습니다.
- **스키마 변경 제한**: 사용자의 명시적 승인 없이 ALTER TABLE, CREATE TABLE 등의 구조 변경을 시도하지 않습니다.
- **읽기 전용 접근**: 모든 소스 코드 분석 시 DB 연결 정보는 '읽기' 전용으로 간주하며, 실제 운영 데이터의 수정을 시도하지 않습니다.

## ⛔ 시스템 및 인프라 보안 (System & OS Security)
docs/os.md에 정의된 서버 환경을 보호하기 위해 다음 행위를 금지합니다.

- **위험 명령어 차단**: 터미널을 통한 rm -rf, format, sudo 권한 획득, 시스템 파일(/etc, /var 등) 수정 시도를 절대 하지 않습니다.
- **환경 변수 보호**: .env 파일이나 설정 파일에 포함된 API Key(Serper, Tavily, Gemini 등) 및 DB 비밀번호를 외부로 노출하거나 문서에 기록하지 않습니다.
- **파일 수정 범위**: 에이전트는 src/ 폴더의 코드를 수정할 수 없으며, 오직 docs/, skills/, research/ 폴더 내의 마크다운(.md) 파일만 생성 및 수정할 수 있습니다.

## ⚠️ 에이전트 거절 프로토콜 (Refusal Protocol)
만약 금지된 행위(예: "DB 테이블 삭제해줘")를 요청받을 경우, 에이전트는 즉시 다음 단계를 수행합니다.

1. **즉시 중단**: 해당 작업을 즉시 멈춥니다.
2. **규정 고지**: 다음과 같이 응답하여 보안 정책을 상기시킵니다.
   > "요청하신 작업은 docs/security.md에 정의된 보안 정책(데이터베이스 보호 및 시스템 수정 금지)을 위반하므로 수행할 수 없습니다."
3. **대안 제시**: 파괴적인 방식이 아닌, 안전한 분석이나 문서화 방향으로 대안을 제시합니다.
