# Astro Blog Skill - 블로그 글 작성 및 이미지 관리 가이드

이 skill은 Astro + GitHub 조합으로 블로그를 운영할 때 지켜야 할 핵심 원칙을 정의합니다.

## 글 작성 규칙

### 1. Frontmatter 필수 포함
모든 블로그 글의 최상단에 아래 Frontmatter를 반드시 포함해야 합니다:

```markdown
---
title: "글의 제목"
date: "YYYY-MM-DD"
description: "글의 요약 설명"
tags: ["태그1", "태그2"] # 선택 사항
slug: "영어-단어-조합" # 선택 사항 (깔끔한 URL을 위해)
category: "기타" # 카테고리 미정 시 "기타" 입력
---
```

### 2. 필수 필드
- **title**: 글의 제목 (문자열) → Astro에서 자동으로 <h1>로 출력됨
- **date**: 발행 날짜 (YYYY-MM-DD 형식)
- **description**: 글의 한 줄 요약 (문자열, SEO를 위해 140~160자 권장)
- **category**: 글의 분류 (필수 기입)

### 3. 선택적 필드 (확장성)
- **tags**: 글의 성격을 나타내는 태그 리스트
- **slug**: 파일명과 관계없이 깔끔한 URL을 위해 사용하는 영어 기반 조합

## 마크다운 구조 및 SEO 최적화

### 1. 본문 시작은 ## (H2)부터
제목(title)이 이미 <h1>로 출력되므로, 본문 내에서 가장 큰 제목은 ##부터 사용해야 합니다.

### 2. 이미지 Alt 태그 필수
모든 이미지(![Alt text](/images/...))에는 반드시 구체적인 설명(Alt text)를 한글로 작성해 주세요.
- ✅ **좋은 예**: `![7곡 제면소 비빔냉면 전체 모습](/images/20260425_183126.jpg)`

## 워크플로우

1. 로컬에서 `src/content/blog/` 디렉토리에 새 `.md` 파일 생성
2. 위 Frontmatter 규칙을 준수하여 작성
3. 본문은 ## 부터 시작하고, 이미지는 적절한 Alt 텍스트와 함께 삽입
4. 이미지는 `/home/swlinux/imgs/`에서 대상 폴더로 복사
5. 파일명은 날짜 기반 생성 (예: `20260430_01.jpg`)
6. 변경 사항을 Git으로 커밋하고 푸시 (자동 배포)
